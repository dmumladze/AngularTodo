YOUR TODO

YOUR TODO is a simple todo list application built by you utilizing the technologies you choose. You are the author, we only have a few requirements that are outlined below.

Step 1: Getting started

First thing we want you to do is to set the application name for the above menu, you can choose to call this whatever you want! Make sure you update the menu and the name in the jumbotron to match!

Step 2: TODO items

Create a dedicated page for viewing and interacting with TODO items.
Expectation:
  • All CRUD based operations should be present.
  • Entry forms with required inputs for all data to be captured.
  • Menu item/link to this new page.
Minimum data to capture:
  • todo [ id, title, description, completed ]

Step 3: TODO filtering

Here we want a page that can show off your skills by filtering on a set of data within a grid/table in some manner.
Expectation:
  • Single text input (button optional) at the top of the page with a grid/table below that is filtered across all displayed columns.
  • If using a button, expecation is to filter when pressing it. Otherwise, if no button, then filter when typing in the text input as the user is typing.
  • Menu item/link to this new page.

FAQ

  • Can we use a 3rd party grid for the filtering and display?
    Answer: Nope, we're not looking for anything fancy, we're more concerned with your ability to simply make it function.
  • Regarding TODO filtering (Step 3) Since we're using MVC, can we filter by posting back to the page?
    Answer: Nope, we're looking specifically for your ability to filter the data utilizing a non-MVC solution. (Hint, hint... Client-side technologies)
  • What about CSS and styling?
    Answer: No specific requirements, however, we would expect good standard practices to be followed.
  • Do we need to worry about authorization?
    Answer: No this can be accessed by anyone that can hit the website.
  • Do I need to use a database?
    Answer: No, this can all be performed by an in-memory data structure of your choosing, just keep in mind anything entered via the CRUD screen (Step 2) should be available for filtering (Step 3).
  • Can I use a database?
    Answer: Yes, provided you are utilizing something that can be rebuilt on a new machine directly from this project, for example Entity Framework code first would be a valid option for this approach.
  • Can I do more than this?
    Answer: Sure, this is your chance to show us what you're capable of, but this is not required.
